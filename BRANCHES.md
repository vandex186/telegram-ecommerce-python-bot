# Ветки проекта (два варианта каталога)

Репозиторий: [wrogg/telegram-ecommerce-python-bot](https://github.com/wrogg/telegram-ecommerce-python-bot)

GitHub хранит код; чтобы бот работал **24/7**, его нужно запустить на сервере (см. [DEPLOY.md](DEPLOY.md)). Ветки ниже — два режима каталога в одном проекте.

## Как сделать форк

1. Откройте репозиторий на GitHub.
2. Нажмите **Fork** → создайте копию в своём аккаунте.
3. Клонируйте **свой** форк:
   ```bash
   git clone https://github.com/YOUR_USERNAME/telegram-ecommerce-python-bot.git
   cd telegram-ecommerce-python-bot
   ```
4. Переключитесь на нужную ветку (см. таблицу).

## Ветки

| Ветка | Название | Описание |
|-------|----------|----------|
| `catalog/inline-buttons` | **Каталог кнопками** | Товары задаются в `config.py` (`PRODUCTS`). В магазине — список inline-кнопок с названиями. Карточка товара — текст и локальное фото (если указано в конфиге). Канал склада не используется. |
| `catalog/channel-cards` | **Каталог выгрузкой карточек** | Товары и цены подтягиваются из приватного Telegram-канала: посты-карточки (описание, фото, форматирование) и отдельные посты с ценами. Кэш последних 30 постов, команды `/sync_last_30`, `/sync_catalog`. |

### Переключение ветки

```bash
# Вариант 1 — кнопки из config
git checkout catalog/inline-buttons

# Вариант 2 — карточки из канала
git checkout catalog/channel-cards
```

После смены ветки: скопируйте `config.example.py` → `config.py`, заполните токен и настройки, затем `./run.sh`.

## Какую ветку выбрать

- **Каталог кнопками** — мало товаров, всё правите вручную в конфиге, без канала.
- **Каталог карточками** — витрина ведётся постами в канале; бот сам обновляет наличие и цены.

## Основная ветка `main`

По умолчанию `main` совпадает с **`catalog/channel-cards`** (актуальная версия с каналом и корзиной).
